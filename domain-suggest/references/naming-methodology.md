# Naming Methodology

Reference for generating and screening startup/company/product/project names. Use alongside `generation-techniques.md`.

## 1. Name Types

Pick a deliberate mix; suggestive, coined, and abstract names are the most defensible and brandable. Descriptive and acronym names are the weakest long-term.

| Type | Description | Examples |
|---|---|---|
| Descriptive | States literally what it does. Clear but hard to own/trademark, limits pivots. | PayPal, General Motors |
| Suggestive / evocative | Hints at a benefit or feeling without describing it (the sweet spot). | Salesforce, Slack, Stripe |
| Invented / coined | Made-up word, no prior meaning; maximally ownable. | Kodak, Spotify, Verizon |
| Abstract / arbitrary | Real word with no logical link; meaning is built by branding. | Apple, Amazon, Oracle |
| Acronym / initialism | From initials; weak unless backed by big spend. | IBM, IKEA |
| Founder / eponymous | Named after a person. | Tesla, Dell |
| Metaphorical | Borrows an image to imply qualities. | Nike, Amazon (river) |
| Compound / portmanteau | Two words joined or blended. | Facebook, Netflix, Pinterest |
| Real-word / lexical | Everyday dictionary word used as-is. | Square, Stripe |
| Foreign-word | Word from another language for meaning/sound. | Audi, Volvo |

## 2. Evaluation Tests

### SMILE - qualities of a strong name (Alexandra Watkins)
- **S**uggestive - evokes something about the brand
- **M**emorable - connects to the familiar
- **I**magery - visual, aids recall
- **L**egs - extends into a theme/brand story
- **E**motional - moves people

### SCRATCH - deal-breakers to avoid
- **S**pelling-challenged (looks like a typo)
- **C**opycat (too close to competitors)
- **R**estrictive (limits future growth)
- **A**nnoying (forced, gimmicky)
- **T**ame (flat, descriptive, uninspired)
- **C**urse of knowledge (only insiders get it)
- **H**ard to pronounce

### Quick checklist
- [ ] Pronounceable on first sight
- [ ] Passes the "radio test" (spellable after only hearing it)
- [ ] Short - ideally 1-3 syllables, few characters
- [ ] Memorable and distinctive vs competitors
- [ ] Positive connotations; no negative meaning in target languages
- [ ] Room to grow (not boxed in by literal meaning)
- [ ] Works written (logo) and spoken (conversation)
- [ ] No hyphens or numbers

## 3. Sound Symbolism (brandability)

Phonemes carry implicit meaning; exploit it to match the brand's feel.
- Front vowel /i/ ("ee") -> small, light, fast, sharp. Back vowels /a/, /o/, /u/ -> large, heavy, solid.
- Plosives /k/, /t/, /p/ -> sharp, energetic (good for tech/edge brands).
- Sonorants /m/, /l/, /b/ -> soft, round, friendly (good for care/comfort brands).
- Phonaesthemes: sub-word clusters carry associations (gl- -> light: gleam, glow). Useful when coining words.
- Aim for 2-3 syllables, clear consonant-vowel rhythm, a strong first phoneme, and a distinctive written silhouette.

## 4. TLD Strategy

- **.com** - default of trust; prioritize for mainstream/enterprise/consumer. If you cannot get the exact `.com` and sell broadly, consider changing the name (Paul Graham's rule). Coined names make exact-match far easier to obtain.
- **.ai** - mainstream for AI startups (Google treats it as generic). Pricier and often a 2-year minimum. Avoid putting "ai" in the name when the zone already says it (redundant).
- **.io / .co** - credible for tech/dev/SaaS audiences; `.co` is weaker and risks leaking traffic to the `.com`.
- **ccTLDs (.ru, .рф, etc.)** - appropriate for region-focused businesses. For `.рф`, names are checked in Punycode.
- **New gTLDs (.app, .dev, .xyz, .tech)** - fine for the right audience; carry mild "not the real one" risk vs `.com`.
- **Zone-aware naming:** when the TLD carries meaning (e.g. `.sale`, `.care`, `.ai`), let the second-level name carry the other signal rather than repeating the zone.

## 5. Legal / Practical (recommend as manual next steps; not done by this skill)

- Trademark: USPTO TESS (US), WIPO Global Brand Database, EUIPO (EU) - check the relevant Nice class.
- Common-law web search for existing unregistered use.
- Social handle availability (X, Instagram, LinkedIn, GitHub, TikTok).
- App store name availability if shipping apps.
- SEO: distinctive coined names "own" their SERP; generic dictionary words face heavy ranking competition.

## 6. Process

1. Brief: positioning, audience, competitors, personality, must-avoids, success criteria.
2. Generate: high volume across techniques (quantity first, no judging).
3. Shortlist: filter against the brief and SMILE/SCRATCH; cluster into themes.
4. Screen: domain availability (via `domain-check`), plus recommend trademark/handle checks.
5. Validate: radio test, say-it-aloud, stakeholder reactions.
6. Select: pick final, secure domain/handles, recommend trademark filing.
